左偏树

左偏树是优先队列的一种实现，支持四种操作：插入、获取最小值，删除最小值，合并两个左偏树。

左偏树是一颗二叉树，每个节点除了键值之外还有距离，距离的定义如下：

当且仅当节点 i 的左子树且右子树为空时，节点被称作外节点（实际上保存在二叉树中的节点都是内节点，外节点是逻辑上存在而无需保存。把一颗二叉树补上全部的外节点，则称为extended binary tree）。节点i的距离是节点 i 到它的后代中的最近的外节点所经过的边数。特别的，如果节点 i 本身是外节点，则它的距离为0;而空节点的距离规定为 -1。

性质：

  1. 节点的键值小于或等于它的左右子节点的键值。

  2. 节点的左子节点的距离不小于右子节点的距离。

  3. 节点的距离等于它的右子节点的距离加1。

  4. 一棵N个节点的左偏树root节点的距离最多为log(N+1)-1。

初始化一个左偏树

初始化左偏树有两种方式。

第一种是每次选择一个节点与树合并，直到所有节点都合并为一个树。这种方法不太有效，时间复杂度为 {\displaystyle O(nlogn)} O(nlogn)。

第二种方法是使用队列，将队列中前两个节点合并，将合并后的新节点放到队列的末尾，直到队列中只有一个节点。这种方法的时间复杂度为 {\displaystyle O(n)} O(n)。

合并两颗左偏树

假设堆是小根堆，合并时选择关键字较小的节点作为根节点，然后将关键字大的节点与根节点的右子堆合并。

在合并之后，比较子堆的s值。通过交换左右子堆来保证左节点的s值始终大于等于右节点。然后更新节点的s值。

Java代码实现合并两棵左偏的最小树：

  1. root键值最小的树的右子树与其它树合并；
  2. 上步合并结果作为与root键值最小树的右子树。
  3. 比较root的左右子树的距离值（s-value），如果右子树大于左子树则交换两棵子树

Fenwick Tree

Fenwick Tree

Fenwick tree 又被称为 Binary Indexed Tree 或者 BIT。

设$f$是可逆函数，$A$是长度为$N$的数组。

BIT有以下性质：

  • 可以在$O(logN)$时间内计算区间$[l, r]$内的$f$的值，即$f(A_l, A_{l+1}, \ldots, A_r)$
  • 可以在$O(logN)$时间内更新$A$的值。
  • 空间复杂度$O(N)$。
  • 实现比较简单。

最常见的应用就是计算区间和，即$f(A_1, A_2, \ldots, A_k)=A_1+A_2+\ldots+A_k$。

为了简单起见，假设$f$是求和。给定一个数组$A[0\ldots N]$，Fenwick Tree就是个数组$T[0\ldots N]$，它的每一项都是$A$在区间$[g(i), i]$的和：$T_i=\sum_{j=g(i)}^iA_j$，其中$0 \leq g(i) \leq i$。

可以写出求和和更新的伪代码：

其中求和函数sum的流程如下:

  1. 把区间$[g(r), r]$的和加到结果中。
  2. 跳到下一个区间$[g(g(r)-1), g(r)-1]$，把和加到结果中。
  3. 持续上述步骤，直到区间$[0, g(g\ldots g(r)-1 \cdots -1)-1)]$的和添加到结果中。

increase的过程是更新$A[i]$的值的时候，需要把所有包含$A[i]$的区间的值都更新。

很明显，两个函数的复杂度都取决于函数$g$，而满足$0 \leq g(i) \leq i$的函数有很多。
例如:

  1. $g(i)=i$，结果就是$T=A$，这样sum函数的复杂度就是$O(N)$，而increase的复杂度是$O(1)$。
  2. $g(i)=0$，得到的$T$就是前缀和，意味着sum的复杂度是$O(1)$，而更新的复杂度变成了$O(N)$。

而Fenwick Tree 巧妙的选择$g(i)$使得更新和求和两个操作的复杂度都变成了$O(logN)$。

Fenwick Tree的$g(i)$ 的操作和简单：用0代替$i$的二进制表示中的末尾的所有的1。
下面是一些例子：

$$
g(11)=g(1011_2)=1000_2=8 \
g(12)=g(1100_2)=1100_2=12 \
g(13)=g(1101_2)=1100_2=12 \
g(14)=g(1110_2)=1110_2=14 \
g(15)=g(1111_2)=0000_2=0 \
$$

实现这个变换也很简单： $g(i) = i & (i+1)$。

现在需要找到一种方法能够遍历所有的满足条件$g(j) \leq i \leq j$的$j$。

很容易发现，可以从$i$开始，不停的把最后一个0变成1就可以了，把这个变换叫做$h(i)$，例如$i=10$时有：

$$
10=0001010_2 \
h(10)=11=0001011_2 \
h(11)=15=0001111_2 \
h(15)=31=0011111_2 \
h(31)=63=0111111_2 \
\vdots
$$
毫不意外得到$h(i) = i | (i+1)$。
