Range Minimum Query

给一个数组 A[1..N], 需要回到查询,在闭区间 [L, R]里面最小的元素是多少.
RMQ可以用于解决LCA问题.

Segment Tree

线段树能够高效的回答数组的范围查询，同时允许更新数组。能够在$O(logN)$时间内查询连续元素$a[l\ldots r]$的和或者最小值。在查询中允许修改元素的值。甚至修改整个子段（例如把$a[l\ldots r]$都设置成某个值，或者都加上一个数）。

总体上来说线段树是非常灵活的数据结构，有很多问题都可以用它来解决。也可以为它增加更复杂的操作，回答更复杂的查询。线段树可以容易的扩展到高维，例如二维线段树可以在$O(log^2{N})$时间内回答子矩阵的和或者最小值。

线段树的一个重要的性质是，它只占用线性的空间，标准的线段树处理长度为$N$的数组需要$4N$的空间。

Segment Tree 的简单形式

有一个数组$a_1, a_2, \ldots, a_n$，线段树能够在$O(logN)$查询区间的和$\sum_l^ra[i]$和更新数组元素$a[i]=x$。

Segment Tree 的结构

构造一颗二叉树，根节点是整个数组的和$\sum_{i=0}^{n-1}a_i$，左子节点是左半部分的和$\sum_{i=0}^{n//2}a_i$，右子节点是右半部分的和$\sum_{i=n//2+1}^{n-1}a_i$。然后对左右半部分继续划分，直到每个节点都是一个元素的和。

下面是数组$a=[1, 3, −2, 8, −7]$的线段树

可以看出线段树的空间复杂度是$O(N)$，总的节点数$=1+2+4+\cdots+2^{\lceil{log_2N}\rceil}=2^{\lceil{log_2N}\rceil+1} \lt 4N$，树的高度是$O(logN)$

构造

递归构造节点，复杂度$O(N)$。

查询

查询给定区间的和$a[l_q\ldots r_q]$，有三种情况：

  1. 查询区间和当前节点的区间一样，$[l_q, r_q] == [l, r]$，直接返回当前节点的值
  2. 查询区间在当前节点的左子树，在左子树中查询
  3. 查询区间和左右子树都相交，在左子树查询$[l_q, (l+r)/2]$，右子树查询$[(l+r)/2+1, r_q]$ 

例如要查询$\sum_{i=2}^4a[i]$，会遍历下图中的浅粉色和浅绿色节点，把浅绿色节点的值加起来得到结果$-2 + 1 = -1$。

下面证明为什么查询复杂度是$O(logN)$，我们可以证明树的每一层最多访问4个节点，而树的高度是$O(logN)$，所以查询复杂度是$O(logN)$。

下面证明每一层最多访问4个节点：

  1. 如果当前层访问的节点数$\leq 2$，那么下一层最多访问4个节点。
  2. 如果层访问了3个或者四个节点，因为查询的区间是连续的，所以中间的节点不会继续向下一层扩展，只有两边的两个节点有可能去下一层查询，最多查询下一层的4个节点。

所以在查询区间和的时候，会去查询已经提前计算好的子区间的结果，时间复杂度是$O(logN)$。

更新

现在考虑更新数组中的某个特定元素，即$a[i] = x$，我们需要重建线段树，使得它和更新后的数组对应。

更新的过程比查询的过程更加简单，因为线段树的每一层都是对数组的一个划分，所以$a[i]$只会影响每一层的一个节点，所以只有$O(logN)$个节点需要更新。

很容易就可以发现，可以使用递归的方式更新线段树。要更新树的当前节点，先更新包含$a[i]$的子节点，然后再更新自己的值。

下图是更新$a[2] = 3$的过程，浅绿色节点是遍历更新过的节点。

实现

首先要考虑如何存储线段树，当然可以使用二叉树结构，但是会存储多余的信息。我们可以使用数组直接存储，根节点存储在下标1，左右节点分别存放在下标2，3。递归的对节点$i$，左右子区间分别存放在$2i, 2i+1$。

Segment Tree 的其他应用

更复杂的查询

  1. 查询最大值、最小值以及最大/小值的个数
  2. 查询最大公约数GCD，最小公倍数LCM
  3. 查询0的个数，第k个0的位置
  4. 查找数组的前缀，使得和大于某个值
  5. 查找具有最大和的子区间，每个节点存储4个值：区间的和，最大前缀和，最大后缀和，子区间内的最大和。{sum, pref, suff, ans}

节点上存储所有的数据

每个节点除了存储上面提到的信息之外，还把整个区间的数据都存起来。空间复杂度是$O(NlogN)$，因为每个值会落到$logN$个节点上。
这样可以解决下面的一些问题：

找到区间内大于给定数字的最小数字

给定三个数字输入$(l, r, x)$，查询区间$[l, r]$里面最小的大于等于$x$的数字。

每个节点存储排序后的当前区间的数字，构建过程只需要$O(NlogN)$时间，因为每个区间排序数组只需要合并两个子节点排好序的数组就行。因为使用了类似meger-sort的方法，所以又被叫做”Merged Segment Tree”。

查询的时候最多访问$O(logN)$的节点，每个节点里面用折半查找最多$O(logN)$次，所以查询的时间复杂度是$O(log^2N)$

找到区间内大于给定数字的最小数字，同时允许更新数组

节点内部不用排序数组存储，而是用红黑树这样的平衡树存储，std::lower_bound查找的复杂度是$O(logN)$，整体的查询复杂度还是$O(log^2N)$。构建时间复杂度是$O(Nlog^2N)$，更新的时间复杂度是$O(log^N)$。

使用”fractional cascading”加速查询

前面的问题使用”fractional cascading”可以在$O(logN)$时间内完成。
前面的查询中，我们把查询分成多个子查询，每个子查询都是用折半查找。使用”fractional cascading” 可以把所有的折半查找替换成一个。
最简单的”fractional cascading”的一个例子：有$k$个排好序的数组，需要从每个数组中找出最小的大于等于给定数字的数字。
我们在排序的数组中，存储{数组，在左子树中的位置， 在右子树中的位置}，我们就不需要在子节点中进行折半查找了。
这种技巧用于没有更新的区间查询，此时的线段树也被称作”Wavelet Tree”。

范围更新(延迟传播 Lazy Propagation)

允许更新一个连续的区间，同时保证查询的复杂度是$O(logN)$。

区间的每个元素都加上一个值

这里我们考虑一个简单的问题，为区间$[l, r]$中的每一个值都加上值$x$，然后查询知道指定位置$a[i]$的值。为了使修改操作更加高效，我们在线段树的每个节点存储需要为整个区间加上的值。
这样只要找到当前节点包含在修改区间内， 我们就不需要再往下查找，需要修改的节点数就从$O(N)$变成了$O(logN)$。

整个区间都更新为指定值

现在要把去买$[l, r]$的所有值都更新成$x$，我们还是考虑查询指定位置的值$a[i]$。
这里主要使用了延迟更新的方式：不更新所有的节点，而是值更新在区间内的根节点，需要一个标记来标记当前的值是否是更新的值。
假设当前节点的值是标记后的值，此时如果要更新它下面的子区间，我们就需要先把之前的值更新到两个子节点。
这里就需要一个push操作，把当前节点的信息更新都子节点中。

区间增加某个值，然后查询区间最大值

解决的方法是在更新和查询之前，先调用push把当前节点的信息传播到两个子节点中。

泛化到更高维度

可以用线段树分割第一个维度，然后每个节点创建一个线段树分割第二维度。

简单二维线段树

给定矩阵$A[0\ldots N-1, 0\ldots M-1]$，需要查询子矩阵$A[x_1\ldots x_2, y_1\ldots y_2]$的和（或者最大最小值），同时可以更新每个元素（例如$A[x][y]=val$）。
构建一个二维线段树先处理$x$再处理$y$，空间复杂度$O(16MN)$，查询时间复杂度$O(logNlogM)$。
